## 为什么 Google 地图抓取需要专业 API

Google Maps 的反爬机制比一般网站狠得多。普通请求发几十次就会触发验证码，IP 直接进黑名单。自己搭代理池维护成本高，而且 Google 的页面是动态渲染的，普通 HTTP 请求拿到的是空壳 HTML。

ScraperAPI 的做法是帮你处理这些脏活：自动轮换 IP、管理请求头、处理 JavaScript 渲染、绕过地理限制。你只需要把目标 URL 丢给它的接口，返回的就是完整渲染后的页面内容。对于 Google Maps 抓取，它还提供了专门的结构化数据端点（Google Maps Results API），直接返回 JSON 格式的商家信息，省去了自己写解析器的麻烦。

## ScraperAPI 定价逻辑：按请求次数计费

ScraperAPI 的计费模型很直白——按 API 请求次数收费，不按数据量、不按带宽。每个套餐给你一定数量的月度请求额度，用完了要么升级，要么等下月重置。

有个细节要注意：Google Maps 这类需要 JavaScript 渲染的请求，每次调用会消耗更多的请求额度（官方标注为每次消耗 10 个 API credits，因为渲染成本更高）。所以在估算成本时，别直接拿套餐里的请求数除以你的目标数据量，要先乘以对应的倍率。
